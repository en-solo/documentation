## Button Component

The Kopi button component is a highly flexible interactive element with multiple variants, types, states, and configuration options for triggering actions and navigation.

### Button Preview

![Button Component](/images/figma/button-component.png)

## Design Specifications

### Spacing

- **Padding (Small)**: 8px top/bottom, 16px left/right
- **Gap**: 8px between icon and label
- **Border Radius**: 16px
- **Height (Small)**: 44px
- **Height (Large)**: Variable based on content

### Colors

**Primary (Default)**
- **Background**: `#17171a` (rgb(23, 23, 26))
- **Text**: `#fcfcfc` (rgb(252, 252, 252))

**Vote For**
- **Default**: `#eeefff` background
- **Active**: `#5861ff` background (purple)
- **Pressed**: `#dedfff` background

**Vote Against**
- **Default**: `#ffeeee` background
- **Active**: `#ff5858` background (red)
- **Pressed**: `#ffdede` background

**Vote Neutral**
- **Default**: `rgba(9, 9, 11, 0.05)` background
- **Active**: `#70707b` background

### Typography

- **Font Family**: Poppins
- **Font Weight**: 400 (Regular)
- **Font Size**: 16px
- **Line Height**: 24px (150%)
- **Letter Spacing**: 0px

### Shadow

Buttons use **no shadow** by default, maintaining a flat, clean appearance.

```css
box-shadow: none;
```

## Component Properties

### Variants

**Primary**
- Main call-to-action style
- Dark background with light text
- Used for primary actions

**Secondary**
- Alternative action style
- Less prominent than primary
- Used for secondary actions

**Ghost**
- Minimal, low-emphasis style
- Transparent or very light background
- Used for tertiary actions

**Vote**
- Special voting button style
- Color-coded for voting interactions
- Types: for, against, neutral

**Special**
- Special authentication buttons
- Google and Apple login variants

### Types

**Default/Inverse**
- Standard dark theme
- Inverted color scheme option

**Vote Types**
- `for` - Positive/approval voting (`#5861ff` active)
- `against` - Negative/rejection voting (`#ff5858` active)
- `neutral` - Neutral voting option (`#70707b` active)

**Special Types**
- `google` - Google authentication
- `apple` - Apple authentication

### States

**Default**
- Normal, ready for interaction
- Base state

**Pressed**
- Actively being clicked
- Slightly darker or lighter shade

**Active**
- Currently selected/active
- Shows current selection in voting

**Disabled**
- Unavailable for interaction
- Reduced opacity, no shadow

### Sizes

**Small (sm)**
- Height: 44px
- Padding: 8px vertical, 16px horizontal
- Most common size

**Large (lg)**
- Larger padding and height
- More prominent actions

### Icon Configuration

**Icon Left**
- Show/hide left icon
- 24px × 24px size
- 8px gap from label

**Icon Right**
- Show/hide right icon
- 24px × 24px size
- 8px gap from label

**Icon Only**
- Display only icon, no label
- Must include `aria-label` for accessibility
- Square 44px × 44px minimum

## Usage Examples

### React/TypeScript

```tsx
import { Button } from '@/components/ui/button';

// Primary button
<Button variant="primary" type="default" size="sm">
  Submit
</Button>

// Secondary with icon
<Button variant="secondary" iconLeft={true}>
  <PlusIcon /> Add Item
</Button>

// Vote for button
<Button variant="vote" type="for" size="sm">
  Vote For
</Button>

// Vote against button
<Button variant="vote" type="against" size="sm">
  Vote Against
</Button>

// Ghost button
<Button variant="ghost" size="sm">
  Cancel
</Button>

// Icon only button
<Button variant="primary" iconOnly={true} aria-label="Close">
  <CloseIcon />
</Button>

// Disabled button
<Button variant="primary" state="disabled" disabled>
  Processing...
</Button>
```

```css
}
```

## Accessibility

### WCAG Compliance

**Keyboard Support**
- `Tab` - Move focus to button
- `Shift + Tab` - Move focus away
- `Enter/Space` - Activate button
- Disabled buttons don't receive focus

**Screen Reader Support**
- Use semantic `<button>` elements
- Include descriptive labels
- For icon-only buttons, always provide `aria-label`
- Use `aria-disabled="true"` for disabled state
- Use `aria-pressed` for toggle-style active states

**Color Contrast**
- Primary: `#fcfcfc` on `#17171a` = 19.8:1 (AAA)
- All variants meet WCAG AA minimum (4.5:1)

**Touch Targets**
- Small size: 44px × 44px ✓ (meets WCAG 2.5.5)
- Adequate 8px spacing between buttons

### ARIA Examples

```html
<!-- Standard button -->
<button type="button">Submit</button>

<!-- Icon-only button -->
<button type="button" aria-label="Close dialog">
  <CloseIcon />
</button>

<!-- Disabled button -->
<button type="button" aria-disabled="true" disabled>
  Submit
</button>

<!-- Active/pressed button -->
<button type="button" aria-pressed="true" class="btn--active">
  Starred
</button>
```

## When to Use

**Primary Actions**
Use `variant="primary"` for the most important action on a page

**Form Submissions**
Submit forms, complete flows, or confirm actions

**Navigation**
Move users to different pages or sections

**Voting Actions**
Use `variant="vote"` with `type="for/against/neutral"` for voting interfaces

## When Not to Use

- **Navigation within text** - Use links instead
- **Too many primary buttons** - Use secondary or ghost for less important actions
- **Non-interactive elements** - Don't style non-clickable elements as buttons
- **Toggle states** - Use toggle switches or checkboxes for binary on/off states

## Best Practices

### Button Hierarchy

✅ **Do:**
- Use one primary button per screen/section
- Use secondary for alternative actions
- Use ghost for tertiary or dismissive actions
- Use vote variant exclusively for voting interfaces

❌ **Don't:**
- Use multiple primary buttons competing for attention
- Mix voting buttons with standard buttons inconsistently

### Icon Usage

✅ **Do:**
- Use icons to enhance button meaning
- Always provide `aria-label` for icon-only buttons
- Use left icons for leading actions, right icons for trailing

❌ **Don't:**
- Use obscure icons without labels
- Overload buttons with both left and right icons
- Use decorative icons that don't add meaning

### States

✅ **Do:**
- Use disabled sparingly with explanatory tooltips
- Use active for currently selected options
- Provide visual feedback for all interactive states

❌ **Don't:**
- Disable without explanation
- Use disabled as the default state
- Rely solely on color to indicate state

## Common Patterns

### Form Actions

```tsx
<div className="form-actions">
  <Button variant="ghost" size="sm">
    Cancel
  </Button>
  <Button variant="primary" size="sm" htmlType="submit">
    Save Changes
  </Button>
</div>
```

### Voting Interface

```tsx
<div className="voting-actions">
  <Button
    variant="vote"
    type="for"
    size="sm"
    state={userVote === 'for' ? 'active' : 'default'}
    onClick={handleVoteFor}
  >
    For ({forCount})
  </Button>

  <Button
    variant="vote"
    type="neutral"
    size="sm"
    state={userVote === 'neutral' ? 'active' : 'default'}
    onClick={handleVoteNeutral}
  >
    Abstain ({neutralCount})
  </Button>

  <Button
    variant="vote"
    type="against"
    size="sm"
    state={userVote === 'against' ? 'active' : 'default'}
    onClick={handleVoteAgainst}
  >
    Against ({againstCount})
  </Button>
</div>
```

### Button with Icons

```tsx
// Left icon
<Button variant="primary" iconLeft={true}>
  <PlusIcon /> Add Item
</Button>

// Right icon (dropdown)
<Button variant="secondary" iconRight={true}>
  Options <ChevronDownIcon />
</Button>

// Icon only
<Button variant="ghost" iconOnly={true} aria-label="More options">
  <MoreIcon />
</Button>
```
