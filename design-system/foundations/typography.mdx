## Font Families

Kopi uses a carefully selected set of four font families, each serving specific purposes in the design system.

### Primary Font

**Font**: `Readex Pro`

**Weights Available**:
- Regular (400)
- Medium (590)
- Semi-Bold (600)
- Bold (700)

**Usage**: Primary typeface for body text, UI elements, and general content

```css
font-family: 'Readex Pro', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
```

### Secondary Font

**Font**: `Poppins`

**Weights Available**:
- Regular (400)
- Medium (590)
- Semi-Bold (600)
- Bold (700)

**Usage**: Secondary typeface for complementary text and UI elements

```css
font-family: 'Poppins', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
```

### Display Font

**Font**: `Lexend`

**Weights Available**:
- Regular (400)
- Medium (590)
- Semi-Bold (600)
- Bold (700)

**Usage**: Large headings, hero sections, and display text

```css
font-family: 'Lexend', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
```

### System Font

**Font**: `SF Pro`

**Usage**: System UI elements, native-looking interfaces

```css
font-family: 'SF Pro', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
```

## Type Scale

### Text Decoration

| Style | Usage | Example |
|-------|-------|---------|
| **Bold** | Strong emphasis, important terms | `font-weight: 590`, `600`, or `700` |
| *Italic* | Subtle emphasis, citations, foreign words | `font-style: italic` |
| Underline | Links (only) | `text-decoration: underline` |
| ~~Strikethrough~~ | Deprecated content, price reductions | `text-decoration: line-through` |

<Warning>
  Avoid using underline for anything other than links to prevent confusion.
</Warning>

## Implementation

### CSS Custom Properties

Use CSS custom properties to implement the Kopi typography system:

```css
:root {
  /* Font Families */
  --font-primary: 'Readex Pro', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
  --font-secondary: 'Poppins', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
  --font-display: 'Lexend', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
  --font-system: 'SF Pro', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;

  /* Font Sizes */
  --text-xs: 0.75rem;    /* 12px */
  --text-sm: 0.875rem;   /* 14px */
  --text-base: 1rem;     /* 16px */
  --text-lg: 1.063rem;   /* 17px */
  --text-xl: 1.25rem;    /* 20px */
  --text-2xl: 1.5rem;    /* 24px */
  --text-3xl: 2.188rem;  /* 35px */

  /* Font Weights */
  --font-regular: 400;
  --font-medium: 590;
  --font-semibold: 600;
  --font-bold: 700;

  /* Line Heights */
  --leading-tight: 0.875rem;   /* 14px */
  --leading-snug: 1rem;        /* 16px */
  --leading-normal: 1.063rem;  /* 17px */
  --leading-relaxed: 1.25rem;  /* 20px */
  --leading-loose: 1.5rem;     /* 24px */
}
```

## Letter Spacing

| Element | Letter Spacing | Purpose |
|---------|---------------|---------|
| Display headings | -0.02em to -0.01em | Tighter spacing for large text |
| Body text | 0 | Normal spacing |
| ALL CAPS text | 0.05em to 0.1em | Improved readability |
| Button labels | 0.02em | Slight spacing for prominence |

## Line Height

<Tip>
  Line height creates vertical rhythm. Use 1.5 (150%) for body text as a baseline for readability.
</Tip>

**Recommended ratios:**
- Headings: 1.1 - 1.3
- Body text: 1.4 - 1.6
- Captions: 1.3 - 1.4

## Usage Guidelines

<Steps>
  <Step title="Establish hierarchy">
    Use the type scale consistently to create clear visual hierarchy. Don't skip levels (e.g., H1 to H3).
  </Step>

  <Step title="Limit font variations">
    Use a maximum of 2-3 font sizes per component or section to maintain simplicity.
  </Step>

  <Step title="Maintain rhythm">
    Use consistent spacing between text elements based on the 8px grid system.
  </Step>

  <Step title="Ensure readability">
    Keep line lengths between 50-75 characters for optimal reading. Use 1.5 line height for body text.
  </Step>

  <Step title="Check accessibility">
    Ensure all text meets WCAG 2.1 AA contrast requirements against its background.
  </Step>
</Steps>

## Responsive Typography

<AccordionGroup>
  <Accordion title="Mobile (< 768px)" icon="mobile">
    Consider scaling down larger text sizes for smaller screens:

    - 3xl (35px): Use 2xl (24px) instead
    - 2xl (24px): Use xl (20px) instead
    - xl (20px): Keep same
    - lg (17px): Keep same
    - base (16px): Keep same (default body)
    - sm (14px): Keep same
    - xs (12px): Keep same
  </Accordion>

  <Accordion title="Tablet (768px - 1024px)" icon="tablet">
    Use the full type scale with minor adjustments:

    - 3xl (35px): May reduce to 2xl (24px) for long headings
    - All other sizes: Use as specified
  </Accordion>

  <Accordion title="Desktop (> 1024px)" icon="desktop">
    Use full type scale as specified:

    - 3xl: 35px
    - 2xl: 24px
    - xl: 20px
    - lg: 17px
    - base: 16px
    - sm: 14px
    - xs: 12px
  </Accordion>
</AccordionGroup>

## Examples

### Using Named Text Styles

```css
/* Display Heading */
.hero-title {
  font-family: var(--font-display);
  font-size: var(--text-3xl);
  font-weight: var(--font-bold);
}

/* Body Text */
.content-text {
  font-family: var(--font-primary);
  font-size: var(--text-base);
  font-weight: var(--font-regular);
  line-height: var(--leading-loose);
}

/* Caption Text */
.caption {
  font-family: var(--font-primary);
  font-size: var(--text-sm);
  font-weight: var(--font-medium);
}
```

### Text Hierarchy

```html
<!-- Display heading with Lexend -->
<h1 class="text-3xl font-bold font-display">Welcome to Kopi</h1>

<!-- Large heading with primary font -->
<h2 class="text-2xl font-semibold font-primary">Getting Started</h2>

<!-- Body text -->
<p class="text-base font-regular font-primary">
  This is default body text using Readex Pro at 16px.
</p>

<!-- Caption with secondary font -->
<span class="text-sm font-medium font-secondary">
  Additional information
</span>
```
