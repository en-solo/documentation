## Overview

The Kopi typography system uses four typefaces to create clear hierarchy and excellent readability across all interfaces. The system includes Poppins for body content, Readex Pro for headings, and SF Pro for system-level elements.

## Typefaces

### Primary: Poppins

Poppins is the main typeface for body text, UI elements, and general content. It provides excellent readability at all sizes.

- **Usage**: Body text, buttons, form labels, UI components, captions, notes
- **Weights Available**: 400 (Regular), 600 (Semi-bold)
- **Character Set**: Latin, Latin Extended
- **Font Family CSS**: `'Poppins', sans-serif`

### Heading: Readex Pro

Readex Pro is used for all heading styles, providing visual distinction and hierarchy.

- **Usage**: Headings, titles, display text, section headers
- **Weight Available**: 400 (Regular)
- **Character Set**: Latin, Latin Extended, Arabic
- **Font Family CSS**: `'Readex Pro', sans-serif`

### System: SF Pro

SF Pro is used for system-level UI elements like status bars on iOS.

- **Usage**: Status bar text, system indicators
- **Weight Available**: 590 (Medium-like)
- **Font Family CSS**: `'SF Pro', -apple-system, BlinkMacSystemFont, sans-serif`

### System Text: SF Pro Text

SF Pro Text is optimized for smaller system text.

- **Usage**: Small system indicators, badges
- **Weight Available**: 600 (Semi-bold)
- **Font Family CSS**: `'SF Pro Text', -apple-system, BlinkMacSystemFont, sans-serif`

## Best Practices

### Do's

✅ Use Readex Pro for all headings
✅ Use Poppins for all body content, buttons, and UI text
✅ Use Body Regular (16px) as your default text size
✅ Use Caption Regular (12px) for small labels and metadata
✅ Use negative letter spacing (-1px) for headings
✅ Maintain the 4px size increments (12px, 14px, 16px, 20px, 24px)
✅ Use line heights that are 133-150% of font size for readability

### Don'ts

❌ Don't use Poppins for headings (use Readex Pro)
❌ Don't use Readex Pro for body text (use Poppins)
❌ Don't use font sizes outside the defined scale
❌ Don't use weights other than 400, 600, and 700
❌ Don't use excessive letter spacing on body text (keep at 0px)
❌ Don't set line heights below 133% (too tight)

## Accessibility

### Minimum Font Sizes

- **Mobile**: 16px minimum for body text to prevent browser zoom
- **Desktop**: 14px minimum for secondary text
- **Never go below 12px** (caption size) for any text

### Line Height

All line heights meet WCAG 2.1 SC 1.4.12 (Text Spacing):
- Body Regular: 150% (24px/16px)
- Body Medium: 143% (20px/14px)
- Note Regular: 143% (20px/14px)
- Caption Regular: 133% (16px/12px)

### Color Contrast

When using typography, ensure text colors meet WCAG AA standards:
- **Body text**: Use `#17171a` (black) on light backgrounds
- **Secondary text**: Use `#26262a` (dark gray) on light backgrounds
- **Tertiary text**: Use `#51515c` (gray) on light backgrounds
