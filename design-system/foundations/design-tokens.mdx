## What are Design Tokens?

Design tokens are the atomic design decisions that make up your design system. They're named entities that store visual design attributes like colors, spacing, and typography values.

<Info>
  Design tokens create a single source of truth for design decisions and make it easy to maintain consistency across platforms and products.
</Info>

## Token Categories

### Color Tokens

Reference colors by their semantic meaning, not their value.

#### Brand Colors

```css
/* Primary Purple */
--color-primary-purple: #9747ff;

/* Primary Blue */
--color-primary-blue: #2684fc;
```

#### Semantic Colors

```css
/* Success */
--color-success: #34c759;

/* Error */
--color-error: #ff5858;

/* Warning */
--color-warning: #ff6900;

/* Info */
--color-info: #5861ff;
```

## Using Design Tokens

### In CSS

```css
.button {
  padding: var(--space-3) var(--space-5);
  background: var(--color-primary-purple);
  color: var(--color-white);
  border-radius: var(--radius-xl);
  box-shadow: var(--shadow-sm);
  font-family: var(--font-family-primary);
  font-size: var(--font-size-base);
  font-weight: var(--font-weight-semibold);
  transition: all var(--duration-base) var(--ease-out);
}

.button:hover {
  background: var(--color-primary-blue);
  box-shadow: var(--shadow-md);
}
```

### In SCSS

```scss
.button {
  padding: var(--space-3) var(--space-5);
  background: var(--color-primary-purple);
  border-radius: var(--radius-xl);
  font-family: var(--font-family-primary);

  &:hover {
    background: var(--color-primary-blue);
  }
}
```

### In JavaScript/React

```javascript
const styles = {
  padding: `var(--space-3) var(--space-5)`,
  backgroundColor: 'var(--color-primary-purple)',
  borderRadius: 'var(--radius-xl)',
  fontFamily: 'var(--font-family-primary)',
}
```

## Semantic Token Aliases

Create semantic aliases for component-specific tokens:

```css
/* Button tokens */
--button-padding-sm: var(--space-2) var(--space-3);
--button-padding-md: var(--space-3) var(--space-5);
--button-padding-lg: var(--space-4) var(--space-6);
--button-radius: var(--radius-xl);
--button-bg-primary: var(--color-primary-purple);
--button-bg-secondary: var(--color-primary-blue);

/* Input tokens */
--input-border-color: var(--color-border-default);
--input-border-color-focus: var(--color-primary-purple);
--input-padding: var(--space-3) var(--space-4);
--input-radius: var(--radius-md);

/* Card tokens */
--card-padding: var(--space-6);
--card-radius: var(--radius-xl);
--card-shadow: var(--shadow-md);
--card-border: var(--color-border-subtle);
```

## Dark Mode Support

Use CSS custom properties to automatically switch values:

```css
:root {
  --color-background: var(--color-background-primary);
  --color-text-primary: var(--color-gray-900);
  --color-text-secondary: var(--color-gray-600);
  --color-border: var(--color-border-default);
}

@media (prefers-color-scheme: dark) {
  :root {
    --color-background: var(--color-gray-900);
    --color-text-primary: var(--color-gray-50);
    --color-text-secondary: var(--color-gray-400);
    --color-border: var(--color-gray-800);
  }
}

/* Or with a class toggle */
[data-theme="dark"] {
  --color-background: var(--color-gray-900);
  --color-text-primary: var(--color-gray-50);
  --color-text-secondary: var(--color-gray-400);
  --color-border: var(--color-gray-800);
}
```

## Token Naming Convention

Follow a consistent naming pattern:

```
[category]-[property]-[variant]-[state]

Examples:
--color-primary-500        (category-property-variant)
--space-4                  (category-variant)
--button-bg-primary-hover  (component-property-variant-state)
```

## Updating from Figma

<Tip>
  When your Figma designs change, update the token values here first, then propagate to your codebase. This keeps documentation and implementation in sync.
</Tip>

<Warning>
  Breaking changes to token names should be versioned and documented to help teams migrate existing code.
</Warning>
