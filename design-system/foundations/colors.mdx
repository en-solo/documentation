## Overview

The Kopi color system uses a dark, minimal palette designed for clarity and focus. The system emphasizes contrast and readability while maintaining a modern, professional aesthetic.

## Primary Colors

### Background Colors

<AccordionGroup>
  <Accordion title="Dark" icon="circle" iconType="solid">
    **Primary Background**

    - **Hex**: `#17171A`
    - **RGB**: `rgb(23, 23, 26)`
    - **Usage**: Main background color, primary surfaces, button backgrounds

    This is the primary dark background used throughout the Kopi interface.
  </Accordion>

  <Accordion title="Light" icon="circle" iconType="solid">
    **Text on Dark**

    - **Hex**: `#FCFCFC`
    - **RGB**: `rgb(252, 252, 252)`
    - **Usage**: Text on dark backgrounds, icons, UI elements

    Near-white color providing excellent contrast on dark backgrounds.
    **Contrast Ratio**: 19.8:1 (WCAG AAA)
  </Accordion>
</AccordionGroup>

## Color Usage

### Interface Colors

The Kopi interface uses a minimal color palette focused on functionality:

| Color Name | Hex Code | Usage |
|------------|----------|-------|
| Background Dark | `#17171A` | Main background, surfaces, button backgrounds |
| Text Primary | `#FCFCFC` | Primary text, icons on dark |
| Background Light | `#FCFCFC` | Light backgrounds (inverse theme) |
| Text Dark | `#17171A` | Text on light backgrounds |

### Semantic Colors

<Note>
Semantic colors for voting and states are defined through type variants in components. See the Button component documentation for voting-specific colors (for, against, neutral).
</Note>

## Accessibility

### Contrast Ratios

All color combinations meet or exceed WCAG AA standards:

| Combination | Contrast Ratio | WCAG Level | Usage |
|-------------|---------------|------------|-------|
| `#FCFCFC` on `#17171A` | 19.8:1 | AAA | Body text, UI elements |
| `#17171A` on `#FCFCFC` | 19.8:1 | AAA | Inverse theme |

<Tip>
**WCAG Requirements:**
- **AA**: Minimum 4.5:1 for normal text, 3:1 for large text
- **AAA**: Minimum 7:1 for normal text, 4.5:1 for large text

All Kopi color combinations exceed AAA requirements.
</Tip>

### Color Blindness

- High contrast ratios ensure visibility for users with color vision deficiencies
- Never rely on color alone to convey information
- Use icons, labels, and text in addition to color coding

## Implementation

### CSS Variables

```css
:root {
  /* Background Colors */
  --color-bg-dark: #17171A;
  --color-bg-light: #FCFCFC;

  /* Text Colors */
  --color-text-on-dark: #FCFCFC;
  --color-text-on-light: #17171A;

  /* Contrast ratios */
  --contrast-primary: 19.8;
}

/* Dark theme (default) */
body {
  background-color: var(--color-bg-dark);
  color: var(--color-text-on-dark);
}

/* Light theme (inverse) */
body.theme-light {
  background-color: var(--color-bg-light);
  color: var(--color-text-on-light);
}
```

### Tailwind CSS

```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        'bg-dark': '#17171A',
        'bg-light': '#FCFCFC',
        'text-primary': '#FCFCFC',
        'text-inverse': '#17171A',
      },
    },
  },
}
```

## Usage Guidelines

### Do's

✅ Use `#17171A` for all dark backgrounds
✅ Use `#FCFCFC` for text on dark backgrounds
✅ Maintain high contrast for readability
✅ Test color combinations for accessibility

### Don'ts

❌ Don't use colors with insufficient contrast
❌ Don't rely solely on color to convey meaning
❌ Don't use too many colors - keep it minimal
❌ Don't reduce opacity below readable levels

## Color Application

### Buttons

- **Primary**: `#17171A` background with `#FCFCFC` text
- **Secondary**: Transparent background with `#17171A` border
- **Ghost**: Minimal styling with `#FCFCFC` text

See the [Button Component](/design-system/components/buttons) for detailed color usage.

### Typography

- **Headings**: `#FCFCFC` on dark backgrounds
- **Body Text**: `#FCFCFC` on dark backgrounds
- **Secondary Text**: `#FCFCFC` with reduced opacity (if needed)

See [Typography](/design-system/foundations/typography) for text color usage.
