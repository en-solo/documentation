## Overview

The Kopi shadow system creates subtle depth and visual hierarchy. Shadows are used sparingly to maintain the clean, minimal aesthetic while providing necessary visual cues for interactive elements.

## Shadow System

Kopi uses minimal shadows focused on functionality rather than decoration.

### Elevation Levels

<AccordionGroup>
  <Accordion title="None" icon="square">
    **No Shadow**

    - **Value**: `none`
    - **Usage**: Flat elements, backgrounds, most text elements
    - **Example**: Default state for most UI elements

    ```css
    box-shadow: none;
    ```

    Most Kopi UI elements have no shadow by default, maintaining a flat, clean appearance.
  </Accordion>

  <Accordion title="Subtle" icon="layer-group">
    **Subtle Shadow**

    - **Offset**: 0px 1px 2px
    - **Color**: `rgba(0, 0, 0, 0.05)`
    - **Blur**: 2px
    - **Usage**: Button default state, subtle elevation

    ```css
    box-shadow: 0px 1px 2px rgba(0, 0, 0, 0.05);
    ```

    Used for buttons and interactive elements in their default state.
  </Accordion>

  <Accordion title="Hover" icon="hand-pointer">
    **Hover Shadow**

    - **Offset**: 0px 2px 4px
    - **Color**: `rgba(0, 0, 0, 0.1)`
    - **Blur**: 4px
    - **Usage**: Button hover state, interactive feedback

    ```css
    box-shadow: 0px 2px 4px rgba(0, 0, 0, 0.1);
    ```

    Applied on hover to provide tactile feedback for interactive elements.
  </Accordion>

  <Accordion title="Elevated" icon="box">
    **Elevated Shadow**

    - **Offset**: 0px 4px 8px
    - **Color**: `rgba(0, 0, 0, 0.12)`
    - **Blur**: 8px
    - **Spread**: 0px
    - **Usage**: Cards, modals, dropdowns

    ```css
    box-shadow: 0px 4px 8px rgba(0, 0, 0, 0.12);
    ```

    Used for elevated surfaces like cards, popovers, and dropdown menus.
  </Accordion>

  <Accordion title="Modal" icon="window">
    **Modal Shadow**

    - **Offset**: 0px 8px 16px
    - **Color**: `rgba(0, 0, 0, 0.15)`
    - **Blur**: 16px
    - **Spread**: 0px
    - **Usage**: Modals, dialogs, overlays

    ```css
    box-shadow: 0px 8px 16px rgba(0, 0, 0, 0.15);
    ```

    Reserved for high-priority overlays that need to stand out above the main content.
  </Accordion>
</AccordionGroup>

## Implementation

### CSS Variables

```css
:root {
  /* Shadow definitions */
  --shadow-none: none;
  --shadow-subtle: 0px 1px 2px rgba(0, 0, 0, 0.05);
  --shadow-hover: 0px 2px 4px rgba(0, 0, 0, 0.1);
  --shadow-elevated: 0px 4px 8px rgba(0, 0, 0, 0.12);
  --shadow-modal: 0px 8px 16px rgba(0, 0, 0, 0.15);

  /* Component-specific */
  --btn-shadow-default: var(--shadow-subtle);
  --btn-shadow-hover: var(--shadow-hover);
  --card-shadow: var(--shadow-elevated);
  --modal-shadow: var(--shadow-modal);
}
```

### Usage Examples

```css
/* Button */
.button {
  box-shadow: var(--shadow-subtle);
  transition: box-shadow 150ms ease-in-out;
}

.button:hover {
  box-shadow: var(--shadow-hover);
}

/* Card */
.card {
  box-shadow: var(--shadow-elevated);
}

/* Modal */
.modal {
  box-shadow: var(--shadow-modal);
}

/* Dropdown */
.dropdown {
  box-shadow: var(--shadow-elevated);
}
```

## Usage Guidelines

### When to Use Shadows

✅ **Do use shadows for:**
- Interactive elements (buttons, cards)
- Hover states for tactile feedback
- Modals and overlays
- Dropdowns and popovers
- Elements that need to appear "above" the page

### When to Avoid Shadows

❌ **Don't use shadows for:**
- Text elements
- Flat backgrounds
- Elements that should blend with the page
- Decorative purposes (keep it functional)
- Too many elements at once (causes visual noise)

## Elevation Hierarchy

Use shadows to create clear visual hierarchy:

1. **Base Layer** (no shadow): Background, most UI elements
2. **Level 1** (subtle): Default buttons, slight interactivity
3. **Level 2** (hover): Hover states, active elements
4. **Level 3** (elevated): Cards, dropdowns, sticky elements
5. **Level 4** (modal): Modals, dialogs, highest priority overlays

## Accessibility

### Color Modes

Shadows work on both dark and light backgrounds:

```css
/* Dark theme (default) */
.element {
  box-shadow: 0px 2px 4px rgba(0, 0, 0, 0.1);
}

/* Light theme - might need adjustment */
.theme-light .element {
  box-shadow: 0px 2px 4px rgba(0, 0, 0, 0.15);
}
```

### Motion Preferences

Respect user preferences for reduced motion:

```css
.button {
  box-shadow: var(--shadow-subtle);
  transition: box-shadow 150ms ease-in-out;
}

@media (prefers-reduced-motion: reduce) {
  .button {
    transition: none;
  }
}
```

## Component Examples

### Button

```css
.button {
  /* Default */
  box-shadow: 0px 1px 2px rgba(0, 0, 0, 0.05);
  transition: box-shadow 150ms ease-in-out;
}

.button:hover {
  /* Hover */
  box-shadow: 0px 2px 4px rgba(0, 0, 0, 0.1);
}

.button:active {
  /* Pressed - reduce shadow */
  box-shadow: 0px 1px 1px rgba(0, 0, 0, 0.05);
}

.button:disabled {
  /* Disabled - no shadow */
  box-shadow: none;
}
```

### Card

```css
.card {
  box-shadow: 0px 4px 8px rgba(0, 0, 0, 0.12);
  border-radius: 12px;
  transition: box-shadow 200ms ease-in-out;
}

.card:hover {
  box-shadow: 0px 6px 12px rgba(0, 0, 0, 0.15);
}
```

### Modal

```css
.modal {
  box-shadow: 0px 8px 16px rgba(0, 0, 0, 0.15);
  border-radius: 16px;
}

.modal-overlay {
  background: rgba(23, 23, 26, 0.8);
  backdrop-filter: blur(4px);
}
```

## Best Practices

### Do's

✅ Use shadows consistently across similar components
✅ Animate shadow changes for smooth transitions
✅ Keep shadows subtle - less is more
✅ Use shadows to indicate interactivity

### Don'ts

❌ Don't use too many shadow levels
❌ Don't make shadows too pronounced
❌ Don't use shadows as primary decoration
❌ Don't forget to test on different backgrounds
