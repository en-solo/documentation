## Overview

The Kopi shadow system uses subtle, layered shadows to create depth and elevation. Shadows are applied sparingly to maintain the clean aesthetic while providing necessary visual hierarchy for cards, headers, and elevated surfaces.

## Shadow System

Kopi uses two primary shadow styles: **Card Shadow** for elevated surfaces and **Header Shadow** for navigation elements.

### Shadow Styles

<AccordionGroup>
  <Accordion title="Card Shadow" icon="layer-group">
    **Elevated Surfaces**

    The Card Shadow uses two subtle shadow layers to create depth for cards and elevated content.

    **Layer 1:**
    - **Type**: Drop Shadow
    - **Offset**: 0px X, 10px Y
    - **Blur**: 10px
    - **Spread**: -6px
    - **Color**: `#09090b` (rgb(9, 9, 11))
    - **Opacity**: 0.1 (10%)

    **Layer 2:**
    - **Type**: Drop Shadow
    - **Offset**: 0px X, 16px Y
    - **Blur**: 16px
    - **Spread**: -6px
    - **Color**: `#09090b` (rgb(9, 9, 11))
    - **Opacity**: 0.1 (10%)

    ```css
    box-shadow:
      0px 10px 10px -6px rgba(9, 9, 11, 0.1),
      0px 16px 16px -6px rgba(9, 9, 11, 0.1);
    ```

    **Used in:**
    - Card component (default, for, against, neutral variants)
    - Elevated surfaces
    - Content containers

    The negative spread (-6px) creates a more refined shadow that doesn't extend too far from the element.
  </Accordion>

  <Accordion title="Header Shadow" icon="window">
    **Navigation Elements**

    The Header Shadow combines two drop shadows with background blur for navigation and header elements.

    **Layer 1:**
    - **Type**: Drop Shadow
    - **Offset**: 0px X, 2px Y
    - **Blur**: 4px
    - **Spread**: 0px
    - **Color**: `#09090b` (rgb(9, 9, 11))
    - **Opacity**: 0.1 (10%)

    **Layer 2:**
    - **Type**: Drop Shadow
    - **Offset**: 0px X, 4px Y
    - **Blur**: 6px
    - **Spread**: -2px
    - **Color**: `#09090b` (rgb(9, 9, 11))
    - **Opacity**: 0.1 (10%)

    **Background Blur:**
    - **Blur**: 8px

    ```css
    box-shadow:
      0px 2px 4px 0px rgba(9, 9, 11, 0.1),
      0px 4px 6px -2px rgba(9, 9, 11, 0.1);
    backdrop-filter: blur(8px);
    ```

    **Used in:**
    - Header component (today, friends variants)
    - Navigation bars
    - Sticky headers
    - Floating navigation elements

    The backdrop blur creates a frosted glass effect when the header overlays content.
  </Accordion>

  <Accordion title="No Shadow" icon="square">
    **Flat Elements**

    Most Kopi UI elements have no shadow by default, maintaining a flat, clean appearance.

    ```css
    box-shadow: none;
    ```

    **Used in:**
    - Buttons (default state)
    - Most text elements
    - Backgrounds
    - Navbar
    - Flat surfaces
  </Accordion>
</AccordionGroup>

## Implementation

### CSS Variables

```css
:root {
  /* Shadow color */
  --shadow-color: #09090b;

  /* Card Shadow - Elevated surfaces */
  --shadow-card:
    0px 10px 10px -6px rgba(9, 9, 11, 0.1),
    0px 16px 16px -6px rgba(9, 9, 11, 0.1);

  /* Header Shadow - Navigation elements */
  --shadow-header:
    0px 2px 4px 0px rgba(9, 9, 11, 0.1),
    0px 4px 6px -2px rgba(9, 9, 11, 0.1);

  /* Background blur for header */
  --header-backdrop-blur: 8px;

  /* No shadow */
  --shadow-none: none;
}
```

### Usage Examples

```css
/* Card component */
.card {
  box-shadow: var(--shadow-card);
  border-radius: 16px;
  background-color: #f9f9f9;
}

/* Header component */
.header {
  box-shadow: var(--shadow-header);
  backdrop-filter: blur(var(--header-backdrop-blur));
  border-radius: 20px;
  background-color: #fefefe;
}

/* Flat button (no shadow) */
.button {
  box-shadow: var(--shadow-none);
  border-radius: 16px;
}
```

## Usage Guidelines

### When to Use Shadows

✅ **Do use shadows for:**
- Card components (elevated content)
- Header/navigation elements
- Elevated surfaces that need to appear "above" the page
- Content containers that need visual separation
- Floating elements

### When to Avoid Shadows

❌ **Don't use shadows for:**
- Buttons (Kopi buttons are flat by default)
- Text elements
- Flat backgrounds
- Navbar component (uses flat background)
- Elements that should blend with the page
- Too many elements at once (causes visual noise)

## Component Shadow Usage

### Card Component

```css
.card {
  box-shadow: var(--shadow-card);
  /* 0px 10px 10px -6px rgba(9, 9, 11, 0.1),
     0px 16px 16px -6px rgba(9, 9, 11, 0.1) */
  background-color: #f9f9f9;
  border-radius: 16px;
  padding: 16px;
}
```

**Variants using Card Shadow:**
- `card-default`
- `card-for` (vote for variant)
- `card-against` (vote against variant)
- `card-neutral` (vote neutral variant)

### Header Component

```css
.header {
  box-shadow: var(--shadow-header);
  /* 0px 2px 4px 0px rgba(9, 9, 11, 0.1),
     0px 4px 6px -2px rgba(9, 9, 11, 0.1) */
  backdrop-filter: blur(8px);
  background-color: #fefefe;
  border-radius: 20px;
  padding: 8px;

```

**Variants using Header Shadow:**
- `header-today`

### Button Component (No Shadow)

```css
.button {
  box-shadow: none;
  background-color: #17171a;
  color: #fcfcfc;
  border-radius: 16px;
  padding: 8px 16px;
}
```

Kopi buttons do not use shadows. They rely on solid backgrounds and colors for visual distinction.

### Navbar Component (No Shadow)

```css
.navbar {
  box-shadow: none;
  background-color: #f3f3f4;
  padding: 24px;
}
```

The Navbar uses a solid background color without shadows.

## Accessibility

### Color Modes

Shadows use `#09090b` (darkest color in the Kopi palette) with low opacity (0.1) to work on light backgrounds.

```css
/* Light theme (default) - works perfectly */
.element {
  box-shadow: 0px 10px 10px -6px rgba(9, 9, 11, 0.1);
}
```

<Note>
If you implement a dark mode, you may need to adjust shadow colors or remove shadows entirely, as dark surfaces typically don't require shadows for depth.
</Note>

### Motion Preferences

Respect user preferences for reduced motion when animating shadows:

```css
.card {
  box-shadow: var(--shadow-card);
  transition: box-shadow 200ms ease-in-out;
}

.card:hover {
  box-shadow:
    0px 12px 12px -6px rgba(9, 9, 11, 0.12),
    0px 18px 18px -6px rgba(9, 9, 11, 0.12);
}

@media (prefers-reduced-motion: reduce) {
  .card {
    transition: none;
  }
}
```

## Elevation Hierarchy

Kopi uses a minimal elevation system:

1. **Base Layer** (no shadow): Background, navbar, buttons, most UI elements
2. **Elevated Layer** (card shadow): Cards, content containers, elevated surfaces
3. **Floating Layer** (header shadow): Headers, navigation, sticky elements with backdrop blur

## Best Practices

### Do's

✅ Use Card Shadow for content cards and elevated surfaces
✅ Use Header Shadow for navigation and header elements
✅ Keep most buttons and interactive elements flat (no shadow)
✅ Use shadows consistently across similar components
✅ Combine Header Shadow with backdrop blur for floating navigation

### Don'ts

❌ Don't use shadows on buttons (keep them flat)
❌ Don't add custom shadow values outside the system
❌ Don't use too many shadowed elements (causes visual clutter)
❌ Don't make shadows too pronounced (keep them subtle)
❌ Don't forget the backdrop blur on header shadows

## Advanced Techniques

### Layered Shadows

Both shadow styles use multiple layers for more realistic depth:

```css
/* Card Shadow uses 2 layers */
box-shadow:
  /* Closer, tighter shadow */
  0px 10px 10px -6px rgba(9, 9, 11, 0.1),
  /* Farther, softer shadow */
  0px 16px 16px -6px rgba(9, 9, 11, 0.1);
```

The two-layer approach creates a more natural shadow that mimics real-world lighting.

### Negative Spread

Using negative spread (-6px in Card Shadow, -2px in Header Shadow) pulls the shadow inward, creating a more refined, less aggressive effect:

```css
/* Without negative spread - shadow extends far */
box-shadow: 0px 10px 10px 0px rgba(9, 9, 11, 0.1);

/* With negative spread - shadow is more refined */
box-shadow: 0px 10px 10px -6px rgba(9, 9, 11, 0.1);
```

### Backdrop Blur

Header components use `backdrop-filter: blur(8px)` for a frosted glass effect:

```css
.header {
  background-color: rgba(254, 254, 254, 0.9); /* Semi-transparent */
  backdrop-filter: blur(8px);
  box-shadow: var(--shadow-header);
}
```

This creates a modern, layered appearance when the header overlays content.
